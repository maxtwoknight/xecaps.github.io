Hello, my name is Max

A small place in the galaxy about me. Nothing spectacular to see, just me.

I've worked at an engineering firm for 20+ years now developing software for engineers and designers to use in their day to day work. I have a degree in Mechanical Engineering Technology and fell into development via getting into things I shouldn't have. Since then, I've gone deeper down the rabbit hole as the years have gone by.

.NET 10+ years

Wish I could say most of my time has been with c#, but alas, it has been vb. I have a few simple wpf and console applications that I support that are c#, but mostly, a large desktop vb application has consumed most of my time.

I've tried to clean up some bad practices that we inherited, but it has been a slow process. I have a couple of projects that use Entity Framework in a very limited way. I did use Dapper in a recent project that I built and was pleasantly surprised at how simple it was to get up and running.

Used sql server constantly for the quite some time now. Daily usage usually involves running queries of some kind. Not so much on administration side. Some time has been spent writing stored procedures. Would like to find a good way of source controlling changes to stored procedures, but my team has continued to struggle with changes.

I have used various tools and languages to access said data such as Access, Excel, Entity Framework, Dapper, Power BI, and Tableau to name a few.

git10+ years

Big proponent of source control and using branches and pull requests. I have had my team somewhat follow the git flow branching model for work, with a few tweaks that work for my team. For personal projects, I usually just a simple quick branch then merge workflow.
